#安装 Mysql 8.0.13

  • 下载 MySQL Community Edition(GPL) 选择不带 Debug 和 Test 的版本（自定）。
  • 解压到自选路径
  • 添加配置文件

解压后这个版本是没有 my-default.ini 的，在根目录下 (eg. D:\Program Files\mysql-8.0.13-winx64\) 新建文本文档 my.ini，内容如下：

[mysql]

# 设置mysql客户端默认字符集

default-character-set=utf8mb4

[mysqld]

#设置3306端口

port=3306 

# 设置mysql的安装目录

basedir=D:\Program Files\mysql-8.0.13-winx64

# 设置mysql数据库的数据的存放目录

datadir=D:\Program Files\mysql-8.0.13-winx64\data

# 允许最大连接数

max_connections=200

# 服务端使用的字符集默认为8比特编码的latin1字符集

character-set-server=utf8mb4

# 创建新表时将使用的默认存储引擎

default-storage-engine=INNODB 

# 默认主机地址（解决navicat连接数据库出现2003...(10061)错误）

bind-address=0.0.0.0

默认字符集推荐 UTF-8MB4，UTF-8 在后面初始化时会出警告(虽然程序员从不看警告\doge)。 端口可以自己设置，不一定要 3306，但不要端口冲突了就行。

安装目录和数据存放目录改成自己的！！！

  • 安装 mysql 服务

    • 以管理员身份运行 cmd 或者 PowerShell，否则安装时会报错。（下面是 PowerShell 操作）

    • cd 到 bin 目录，比如：cd d:\Program Files\mysql-8.0.13-winx64\bin\

    • 初始化 mysqld --initialize --console (如果没有为 mysql 添加环境变量，就是 .\mysqld --initialize --console)，等待一段时间，在根目录下的 data 文件夹里，就会生成一堆文件。

      控制台里会给一些提示，其中 A temporary password is generated for root@localhost: 后面跟的是随机生成的初始登录密码，复制备用。

    • 安装服务 mysqld -install，成功或者已经存在都会有提示。移除是 mysqld -remove

    • 启动服务 net start mysql，没有初始化的话，这里会出现(3534)错误。停止服务是 net stop mysql

    • 登录 mysql -u root -p，之后回车输入初始密码，然后就登录成功了。可以测试 show databases; 退出键入 quit

    • 修改密码 ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'xxxxxxx';xxxxxxx是要设的密码。
